Ruby Hashes vs Javascript Objects

The Comparison is ON!!

November 16, 2015


Ruby Hashes (Literal Constructor) 

          
            ruby_hash = {

              "key 1" => "I'm key 1's value!",
              "key 2" => "You ain't that special key 1 cuz I'm key 2's value",
              "key 3" => "Calm down guys"

            }

We have just created a hash in Ruby using the literal constructor method, with three keys (as strings) and their corresponding values by using the "=>" syntax. To access the value of say the first key, we'd write ruby_hash["key 1"] . To change the value, all we'd have to do is set ruby_hash["key 1"] equal to something else like ruby_hash["key 1"] = "I am the best cuz I'm numba one". Simple right? Let's check out how storing info in a somewhat similar fashion works in Javascript.


Javascript Objects (Literal Constructor) 

          
            javascriptObject = {

              key1: "I'm key1's value",
              key2: "I respect you key 1",
              key3: "chicken noodle soup"

            };

There seems to be a bit more respect amongst the keys here in Javscript's object. In Javascript, however, keys are more often referred to as 'properties' of an object. To access the value of a property, we'd use dot notation, like so javascriptObject.key1 . Changing the value would be the same as in Ruby, where we'd set javascriptObject.key1 equal to something else, like javascriptObject.key1 = "pizza shaped cloud" .


Debrief

I have very quickly illustrated the main syntax differences between a Hash in Ruby and an Object in Javascript using the Literal Constructor method, but the functions/uses of both are fundamentally the same, as in they both store information in the form of key/value pairs.
