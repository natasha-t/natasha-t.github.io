Arrays 'n Hashes

A place to store your objects...how convenient!

October 23, 2015


Ar-r-rays!

As you've continued your way through your precious intro to programming journey you must have come across arrays by now because how can you not? It just seems to make a lot of sense for there to be an option to store data in a list format, where items are separated by commas. And that is precisely what an array is. Say you have a group of your most prized posessions. Right now in your program you have each possession saved in an in its own variable. You may have something like this: 

possession_1 = "Latte-stained Moleskine"
possession_2 = "Monster's Inc Boo Doll"
possession_3 = "Japanese Food Erasers"


You could keep going forever! But having these possesions laid out like this in the program doesn't allow you to do that much with them. What if you wanted to add more possesions? Or remove some? Or have each of those possessions do something, like get sorted alphabetically, all in one quick motion? Kinda hard to do all that with your current list of variables. What would be ideal is to group these possesions togethr into one entity. Arrays to the rescue! All you've gots to do is plop those possessions into an array and you have a very nifty and easily accessbile collection of posessions. Let's name your array, like mah_possessions, and place them between two brackets [], also known as the literal constructor. Now you have 

mah_possessions = [possession_1, possession_2, possession_3]

Coolio, right?. But how do you access each possesion? Easy. Since an array is a linear and ordered list of objects (in this case, strings or text), each item in the array is indexed. The starting index value is 0, not 1 like you may think, but 0 because, well, computers. To get at possession_1, you simply type mah_possessions[0] and voila the computer hands you your latte-stained Moleskine. As you can guess, to get possession_2 and possession_3 you type mah_possessions[1] and mah_possession[2] . What you've gotta do now is check out the Ruby docs for array manipulation because it's quite remarkable what you can continue doing with arrays in Ruby. It's like a treasure-trove of methods, like .push to add to your possessions array, or .pop to remove some. Check them all out here.


Hash [browns]

Ok so turns out Hashes in Ruby are not perfectly golden fried potatoes but actually another way to house a collection of objects in your program. Though that would be pretty awesome if everytime you made a new hash in Ruby your computer ejected a fresh, warm hash brown for your enjoyment; you could even program an option to cook tater tots too, because tater tots are kinda like chopped up hash browns right? Unfortunately computers and programming languages can't do that (YET) so the use of hashes in Ruby will have to be limited to a form of storage for your program's objects. Well actually that's not limiting at all; hasehs are, in fact, highly capable container objects that are much more dynamic in terms of storing data than arrays.

The structure of a hash consists of what's called key/value pairs. Keys are like labels for the values you want to be in the hash. There can only be one type of key per hash, but you can of course have duplicate values for any given key. To make this less abstract-sounding lets go back to our mah_possessions array. Say we want to expand on this information and transform the data into a hash. We could name our new hash mah_possessions_hash and begin by populating its contents in between two curly brackets {} (literal constructor). Your mah_posessions_hash object would look something like this: 

mah_possessions_hash = {
possession_1: "Latte-stained Moleskine",
possession_2: "Monster's Inc Boo Doll",
possession_3: "Japanese Food Erasers"
}


As you can see the possession number and its associated value are paired together in a hash via a colon. It's great how clear this information is laid our for you. To access a hash value, you have to use the key it's associated with (as opposed to an array's index value). So to get "Japanese Food Erasers" you'd write mah_possessions_hash[:possesion_3] (note that the colon goes before the key name). Hashes can contain as many key/value pairs as you want, and also allow for a host of manipulation which can be accessed here.
